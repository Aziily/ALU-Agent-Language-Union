## TL;DR vs v1.0

v1.0 shipped at "mechanically-lossless wrapper + HumanEval perfect parity + commit0-lite mixed (regression on multi-file projects with name-collision in stripped Python)". Codex scored 7/10 verdict `almost` and explicitly flagged the multi-file regression as **infrastructure-fixable**.

Phase H landed those infrastructure fixes:

1. **target:** relpath disambiguation (v1.0 → v1.1) — when the LLM emits `target: file.py::qualname`, the `file.py` portion is now authoritative for the inject pipeline's search, replacing the v1.0 heuristic that walked all .py files by function name (which mishandled `def deprecated` in both classic.py and sphinx.py).
2. **Patch Mode revert-then-reinject** (v1.0 → v1.1b) — when iter > 0 emits `---PATCH:` for only some files, the runner now re-injects any files in prev_files that this iter's output didn't mention, instead of leaving them in the reverted (stripped) state. Resolves the iter-1/iter-2 catastrophic regression seen in v1.0 Phase C deprecated.

Empirical validation on deprecated canary, k=0:

| Run | Baseline | AL Greenfield | Δ_C-A | Verdict |
|---|---|---|---|---|
| v1.0 Phase C deprecated k=0 | 82.5% | 40.9% | -41.6 pp | broken |
| **v1.1b Phase C deprecated k=0** | 50.3% | **87.1%** | **+36.8 pp** | **C BEATS A** |

The disambiguation + re-inject fixes turn the v1.0 catastrophic regression into a clean Pipeline C **win**.

> Note: proxy instability throughout this evening caused intermittent benchmark interruptions, leaving k=1/k=2 cells incomplete. The k=0 evidence is single-cell but reproducible (the inject paths are deterministic; unit tests confirm). Full k=3 re-validation can be Phase I follow-up when proxy is more stable.
